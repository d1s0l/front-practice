# Value Quest

Value Quest — это браузерная игра на Next.js, в которой игрок перемещается по полноэкранной сцене, подходит к NPC, выбирает реплики в диалогах и влияет на метрики командных ценностей: репутацию, ответственность, прозрачность, скорость и качество.

Сейчас в проекте уже есть:
- полноэкранная игровая карта со сценой на `@react-three/fiber` и `@react-three/drei`
- игрок и интерактивные NPC-зоны
- интерактивный кот-компаньон с 3D-моделью
- диалоговый flow с несколькими этапами
- arcade pong mini-game у отдельного NPC
- HUD с прогрессом, метриками и таймером
- отдельные mission pages для каждой ценности
- лёгкий mock AI-слой, который генерирует диалоги, подсказки и сопроводительный текст

## Игровой цикл

1. Игрок перемещается по основной сцене с клавиатуры или через mobile controls.
2. Когда игрок входит в радиус взаимодействия NPC, появляются CTA-кнопка и контекстная плашка.
3. Открытие диалога запускает текущий этап NPC и, если нужно, таймер этапа.
4. Для NPC `Макс` сначала запускается быстрый пинг-понг раунд с bonus XP.
5. Для NPC `Соня` сначала запускается arcade pong challenge с отдельным счётом, AI-соперником и управлением ракеткой.
6. Каждый выбранный ответ меняет профиль метрик и либо продвигает игрока дальше, либо предлагает повторить попытку.
7. При подходе к коту можно нажать `E`, погладить его и увидеть burst сердечек над моделью.
8. После завершения всех NPC-веток открываются итоговый summary overlay и экран с обратной связью.

## Взаимодействие с NPC

- Конфигурация NPC находится в [shared/config/game-npcs.ts](shared/config/game-npcs.ts).
- Логика определения ближайшего NPC находится в [useZoneInteraction.ts](features/zone-interaction/model/useZoneInteraction.ts).
- Сессия игры, очки, переходы между этапами, retry-логика и финальный flow управляются в [useGameSession.ts](processes/game-session/model/useGameSession.ts).
- Основной UI диалога отрисовывается в [DialogueModal.tsx](features/dialogue/ui/DialogueModal.tsx).
- Для NPC `Макс` добавлен быстрый режим вопросов-ответов `rapid sync`, который открывается перед основным диалогом и даёт bonus XP за точные и быстрые ответы.
- Для NPC `Соня` добавлен отдельный arcade pong challenge, который открывается перед обычным диалогом, играет как самостоятельный матч до 3 очков и после завершения возвращает игрока в комнату.

## Рейтинг и прогресс

- `reputation`, `responsibility`, `transparency`, `speed`, `quality` и `xp` изменяются на основе выбранного ответа.
- Уровень игрока вычисляется из XP внутри [useGameSession.ts](processes/game-session/model/useGameSession.ts).
- Прогресс в HUD считается по числу завершённых NPC-веток относительно общего количества NPC.
- Для timed stages могут применяться штрафы по timeout, если игрок не ответил вовремя.
- Для quick ping-pong mini-game начисляется дополнительный XP: базовый за попытку, бонус за правильный ответ и дополнительный бонус за быстрый ответ.
- Для arcade pong начисляется XP за участие `+10`, дополнительный XP за победу `+22` и отдельный flawless-бонус `+12` за матч без пропущенных мячей.

## AI-слой

Сейчас проект использует не живую модель, а mock AI-хелпер. Функции из [cursor-ai.ts](shared/lib/ai/cursor-ai.ts) генерируют:

- заголовок и сопроводительный текст диалога
- превью-копирайт для NPC-квеста
- контекстные подсказки после выбора ответа
- outcome text после завершения этапа

Это позволяет сохранить детерминированное поведение UI и игрового flow, но при этом оставить ощущение AI-слоя в подаче.

## 3D-слой

3D-часть проекта построена на связке `@react-three/fiber` + `@react-three/drei`.

Сейчас в сцене используются:
- `Canvas` из `@react-three/fiber` как контейнер 3D-мира
- `PerspectiveCamera` из `@react-three/drei` для follow-camera
- `RoundedBox` для более декларативной сборки игрока, NPC и части окружения
- `Float` для активных маркеров NPC
- `ContactShadows` для общей контактной тени сцены
- `Preload` для предварительной загрузки scene-assets внутри canvas

Переведённые части:
- камера сцены
- геометрия игрока
- геометрия NPC
- основные платформы и границы карты
- общая организация 3D-сцены
- загрузка кота из `public/cat.glb`
- декор стен с IT-постерами
- ретро-эффект сердечек над котом

При этом UI-слой по-прежнему отделён от 3D:
- HUD
- диалоги
- CTA-кнопки
- модалки
- overlay-элементы

Новые scene-элементы:
- кот-компаньон `Пиксель` в одном из углов комнаты
- heart burst при взаимодействии
- расширенная комната с дополнительными платформами
- IT-постеры на стенах через `Html` из `drei`
- arcade ping-pong corner рядом с NPC скорости
- arcade pong overlay с отдельным HUD, счётом и мобильными кнопками `вверх / вниз`

## Адаптивный интерфейс

Основной UI игры рассчитан на fullscreen-режим и теперь включает явное поведение для больших экранов:

- контролируемые ширины и высоты модалки через `clamp()` и max constraints
- внутренний scroll только в контентной области диалога, когда контента действительно много
- отдельные размеры для HUD, interaction prompt и финальных overlay на больших экранах
- безопасный перенос длинных строк и текстов в кнопках
- fullscreen overlay layout без page-level горизонтального скролла

Ключевые стили находятся в:

## Структура проекта

Кодовая база следует практичному FSD-подходу:

- `app` — маршруты Next.js и корневой layout
- `processes` — orchestration-логика и многошаговый flow игры
- `widgets` — крупные игровые блоки вроде главного хаба, HUD и overlay
- `features` — диалоги, движение, решения и взаимодействие
- `entities` — типизированные доменные данные и небольшие доменные UI-компоненты
- `shared` — конфигурация, утилиты и общие стили

## Примечания

- Three.js работает только на клиенте через React Three Fiber и drei-утилиты.
- По умолчанию проект опирается на server components, а client components используются только там, где нужна интерактивность или WebGL.
- Для локальной разработки не нужен ни live backend, ни реальная AI-интеграция.
- Шрифты переведены на локальные CSS font stacks, поэтому сборка не зависит от внешней загрузки Google Fonts.
